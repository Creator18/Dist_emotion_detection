# Distributed Emotion Detection System

A distributed framework for emotion detection using independent binary classification models for each emotion (Anger, Joy, Sadness, Surprise, Fear). Each model runs in its own Docker container, coordinated by a central orchestrator that aggregates predictions and identifies both primary and secondary emotions with confidence scores.

## Prerequisites

- Python 3.8+
- Docker Desktop (download from https://www.docker.com/products/docker-desktop/)
- pip packages for the orchestrator: `flask`, `requests`
- pip packages for training: `torch`, `transformers`, `scikit-learn`, `pandas`, `numpy`, `nltk`

## Step 1: Training the Models

If you don't already have the `.pth` model files and tokenizer folders, you need to train them first.

1. Place your synthetic dataset CSV (e.g., `unique_synthetic_joy_dataset.csv`) in the training directory.
2. Open `train_model.ipynb` and run all cells.
3. For each emotion, update these variables before running:
   - `file_path` вЂ” path to that emotion's CSV dataset
   - `emotion` вЂ” the target emotion name (e.g., `'anger'`, `'joy'`, `'sadness'`, `'surprise'`, `'fear'`)
   - `model_save_path` вЂ” e.g., `"anger_classification_model.pth"`
   - `tokenizer_save_path` вЂ” e.g., `"anger_tokenizer"`
4. After training, move each `.pth` file and tokenizer folder into the corresponding node folder.

**Note:** The trained model files (`.pth`) are not included in this repository due to their large size. They must be generated by running the training notebook, or downloaded separately if hosted externally.

## Step 2: Setting Up Docker

1. Install and open Docker Desktop. Ensure the engine is running (whale icon in system tray).
2. On Windows, Docker Desktop may prompt you to enable WSL 2 during installation вЂ” follow those instructions.

## Step 3: Building and Running the Nodes

Open a terminal and build each emotion node:

```bash
cd anger_node
docker build -t anger-node .
docker run -d -p 8001:8000 anger-node

cd ../joy_node
docker build -t joy-node .
docker run -d -p 8002:8000 joy-node

cd ../sadness_node
docker build -t sadness-node .
docker run -d -p 8003:8000 sadness-node

cd ../surprise_node
docker build -t surprise-node .
docker run -d -p 8004:8000 surprise-node

cd ../fear_node
docker build -t fear-node .
docker run -d -p 8005:8000 fear-node
```

Verify all containers are running with:

```bash
docker ps
```

You should see five containers on ports 8001вЂ“8005.

## Step 4: Running the Orchestrator

From the project root directory:

```bash
pip install flask requests
python orchestrator.py
```

Open your browser and go to `http://127.0.0.1:5000`. Enter a sentence and click "Predict Emotion" to see the primary and secondary emotion results.

## Useful Docker Commands

- `docker ps` вЂ” list running containers
- `docker logs <container_name>` вЂ” view container logs
- `docker stop <container_name>` вЂ” stop a container
- `docker rm <container_name>` вЂ” remove a stopped container

## Tech Stack

- RoBERTa (transformer-based binary classifiers)
- FastAPI (emotion node servers)
- Flask (orchestrator and web interface)
- Docker (containerization)
- Blockchain-inspired consensus mechanism (prediction validation)
